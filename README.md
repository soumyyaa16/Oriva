# Oriva — Prescription Intelligence & Safety Engine

> A DBMS-driven clinical decision support system that detects dangerous drug interactions, computes AI-inspired patient risk scores, and recommends cost-effective alternatives integrated directly into the hospital prescription workflow.

![MySQL](https://img.shields.io/badge/MySQL-8.0-blue) ![Python](https://img.shields.io/badge/Python-3.14-green) ![Flask](https://img.shields.io/badge/Flask-3.1-lightgrey) ![Status](https://img.shields.io/badge/Status-Live-brightgreen)

## The Problem

Indian hospitals currently check drug interactions manually using memory or standalone websites like Drugs.com with no integration into the patient's prescription history. Studies show over 67% of prescriptions with 3+ drugs contain at least one clinically significant interaction. Patients also face avoidable pharmaceutical costs due to zero cost awareness at the point of prescription.

## What Oriva Does

- Automatically detects dangerous drug-drug interactions when a prescription is added
- Computes a live clinical risk score for every patient using a rule-based AI formula
- Recommends safer and cheaper generic alternatives with exact cost savings
- Maintains a full audit trail of all alerts and prescriptions
- Deployed as a live web application accessible from any device

## Database Architecture

7 core tables + 1 dynamic VIEW:

- **Patients** — demographics and clinical parameters
- **Doctors** — prescribing physicians
- **Drugs** — 20 drugs with brand + generic pricing
- **Prescriptions** — links patients, doctors, and drugs
- **Drug_Interactions** — 15 clinically verified dangerous combinations
- **Alternatives** — safer and cheaper substitutes
- **Alerts** — auto-generated by TRIGGER on dangerous prescriptions
- **Patient_Risk_View** — live AI risk scoring (computed, never stored)

---

## The Three Layers

### DBMS Core
Relational schema with full referential integrity, foreign keys, and ACID-compliant transactions.

### AI Layer
Rule-based clinical risk scoring formula:
risk_score = (age × 0.3) + (comorbidities × 20) + (100 − oxygen_level)


Computed dynamically via SQL VIEW never hardcoded.

### Finance Layer
Every interaction warning includes a generic alternative suggestion with live cost savings calculated using a custom SQL FUNCTION.
